# 安全工程交付物规范

安全工程师是**最后一道防线**，对越权、字段泄露、审计可追溯负责。

## 核心职责

1. 维护越权用例库（水平/垂直/跨租户）
2. 审核字段级安全（FLS）规则配置
3. 审计日志的完整性与防篡改
4. 敏感变更的二次确认与审批流
5. 定期红蓝对抗，输出漏洞报告

## 越权分类与用例清单

### 水平越权（同级用户互相访问）

```
- [ ] A 租户用户访问 A 租户其他用户的订单详情 → 403
- [ ] 同租户不同部门用户按 ID 直接访问对方资源 → 403（数据范围之外）
- [ ] 修改/删除他人资源 → 403
- [ ] 导出接口不受"数据范围"约束的漏洞 → 覆盖
- [ ] 批量接口中混入他人 ID（如 ids=[own, others]）→ 整体拒绝
```

### 垂直越权（普通用户 → 管理员）

```
- [ ] 普通用户调用管理员接口（按 URL 猜测）→ 403
- [ ] 通过修改前端隐藏按钮的响应数据直接 POST → 403
- [ ] 通过伪造 Header / Claim 提升权限 → Token 校验拒绝
- [ ] 超管接口限制 IP 白名单 / MFA
```

### 跨租户越权（最严重）

```
- [ ] B 租户用户构造请求访问 A 租户数据 → 403 且审计
- [ ] 服务间调用伪造 X-Tenant-Id Header → 拒绝（需 mTLS + 白名单）
- [ ] 分享链接 / 导出文件跨租户可访问 → 失败
- [ ] 缓存命中串数据（key 不含 tenantId）→ 测试通过
- [ ] 消息队列消费错误租户数据 → 幂等 + 拒绝
```

## FLS 规则审核

基于 `SysFieldLevelSecurity`：

| 审核项 | 要求 |
|--------|------|
| 目标维度 | 角色/用户/部门至少一项非空 |
| 策略选择 | 与字段性质匹配：手机号→PartialMask，密码→Deny，身份证→Hash/Encrypted |
| 生效顺序 | Deny > FullMask > Hash > Encrypted > PartialMask > None（**严格优先级**） |
| 覆盖面 | 列表 / 详情 / 导出 / 打印 四个出口均覆盖 |
| 例外 | 白名单用户/角色需单独审批并记录 ADR |

### 敏感字段分级

| 级别 | 示例 | 默认策略 |
|------|------|----------|
| L1 公开 | 用户名昵称 | None |
| L2 内部 | 邮箱、手机号 | PartialMask |
| L3 敏感 | 身份证、银行卡 | Hash/Encrypted |
| L4 高敏 | 密码、密钥、Token | Deny |
| L5 合规 | 医疗记录、未成年信息 | Encrypted + 审批解密 |

## 审计完整性

审计日志必须满足：

- **只追加**：数据库权限仅 `INSERT`，无 `UPDATE/DELETE`
- **时间可信**：服务端时间戳，NTP 同步，允许时间偏移 < 2s
- **链路可追溯**：`CorrelationId` 贯穿前端 → 网关 → 服务 → 审计
- **内容完整**：`Before/After` 对敏感变更记录 JSON 快照（同样经 FLS 脱敏后存储）
- **防篡改**：定期计算哈希链（前一条 hash + 当前内容 → 当前 hash）
- **归档与备份**：冷数据对象存储 + 跨区备份，保留 ≥ 6 个月

## 敏感操作二次确认

在以下操作处必须二次确认 + 记录原因：

- 批量导出（> 阈值行数）
- 跨租户查询（平台超管）
- FLS 白名单临时解密
- 删除/归档审计日志
- 变更 MFA / 手机号 / 支付密码
- 权限/角色的授予与撤销（尤其超管授予）

UI 模式：**输入原因** + **输入当前密码 / MFA 验证码** → 提交。

## 红蓝对抗清单

```
- [ ] 每季度一次内部越权演练
- [ ] 每半年一次字段泄露演练（含导出/打印/日志）
- [ ] 权限变更事件回放验证（是否真实失效缓存）
- [ ] JWT 泄漏/重放场景演练
- [ ] 服务间鉴权绕过演练
- [ ] 对接 SRC / 外部白帽，定期披露修复
```

## 与 DevOps 的协作

- 生产审计库独立账号，DBA 只读、审计员只写、无人可改
- 日志脱敏在采集端完成（Filebeat / OpenTelemetry processor）
- 越权告警实时推送 IM（阈值：同一用户 1 分钟内 3 次 403 跨租户）
- 发布前必过"权限 Diff 审计"：新旧权限集合变化报告

## 安全评审清单

```
- [ ] 越权三类用例全覆盖（水平/垂直/跨租户）
- [ ] FLS 规则经安全审核，例外有 ADR
- [ ] 审计日志完整、不可篡改、可追溯
- [ ] 敏感操作二次确认与原因记录
- [ ] JWT 含 tid/uid，签名算法 ≥ RS256，过期 ≤ 2h
- [ ] 服务间调用 mTLS + Header 白名单
- [ ] 生产日志经脱敏后采集
- [ ] 告警阈值与通道已验证
```

## 常见安全反模式

- ❌ 把审计写到业务库，可被业务账号 UPDATE
- ❌ FLS 依赖前端实现（F12 一看原值）
- ❌ 用 `/api/admin/*` 前缀做权限判定（路径可被篡改）
- ❌ 平台超管账户无 MFA / 无操作审计
- ❌ JWT 过期时间 > 24h 且无刷新机制
- ❌ 导出接口绕过列表 DTO 直接走原始实体
- ❌ 日志中明文打印手机号/身份证
