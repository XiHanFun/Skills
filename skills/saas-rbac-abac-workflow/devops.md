# DevOps 交付物规范

DevOps 保证权限 / 租户变更**可灰度、可观测、可回滚**。

## 核心职责

1. 租户级配置与特性开关（Feature Flag）
2. 数据库迁移编排与回滚演练
3. 灰度发布与蓝绿部署
4. 监控告警：授权失败率、隔离违规、审计写入
5. 生产数据访问合规（DBA 审批 / 堡垒机 / 操作录像）

## 租户配置模型

使用本仓库的 `SysConfig`：

| 作用域 | 示例键 | 说明 |
|--------|--------|------|
| 全局 | `auth.jwt.expiration` | 所有租户默认值 |
| 租户 | `feature.bulk_export.enabled` | 单租户覆盖 |
| 用户 | `ui.density` | 最细粒度（非权限相关） |

规则：
- 配置变更走发布流水线，禁止手工改生产 DB
- 租户级配置变更必须审批 + 审计
- 敏感配置（密钥/连接串）走密钥管理（Vault / KMS），不入 SysConfig

## 特性开关（Feature Flag）

```
feature.rbac.new_policy_engine.enabled
feature.abac.field_masking_v2.enabled
feature.tenant.isolation.physical_db.enabled
```

使用规则：
- 每个新权限/ABAC 能力都要挂 Flag，默认 **关闭**
- 白名单先行：内部租户 → 灰度租户 → 全量
- Flag 生命周期 ≤ 3 个月；过期必须清理或转为永久配置

## 灰度发布流程

```
阶段 0：内部租户（公司自用）验证 1 ~ 2 天
阶段 1：灰度租户（预先签约自愿参与）5% 3 天
阶段 2：扩量到 25% 3 天
阶段 3：50% 3 天
阶段 4：全量
```

每阶段门禁：
- 授权失败率 < 基线 + 0.5%
- p99 延迟 < 基线 + 10%
- 零跨租户告警
- 零 P0/P1 事故

任一不达标：**自动回滚**到上一阶段。

## 发布流水线关卡

```
- [ ] 单元 + 集成测试绿
- [ ] @tenant-isolation 标签用例 100% 通过
- [ ] 权限 Diff 报告已产出并审批
- [ ] DB 迁移脚本含 up/down/verify
- [ ] 回滚脚本在 staging 演练通过
- [ ] 特性开关默认关闭
- [ ] 监控面板已添加相关指标
- [ ] 变更通知已发送给对应业务方
```

## 数据库迁移编排

1. **兼容先行**：新旧字段并存，双写一段时间
2. **部署新代码**：读旧或读新由 Flag 控制
3. **数据回填**：按租户分批，夜间窗口
4. **切读**：Flag 逐批切到读新
5. **下线旧**：确认零读后删除旧列/表

禁止"一次性 DDL + 代码上线"的紧耦合发布。

## 监控与告警

### 必备指标

| 指标 | 采集 | 阈值（示例） |
|------|------|--------------|
| 授权判定 QPS | 中间件 metrics | - |
| 授权判定 p99 | histogram | > 10ms 告警 |
| 授权失败率（403） | counter | 超基线 +1% 告警 |
| 跨租户访问拒绝数 | counter | > 0 即告警（P1） |
| 权限缓存命中率 | gauge | < 90% 告警 |
| 审计写入延迟 | histogram | p99 > 5s 告警 |
| 审计写入失败数 | counter | > 0 告警 |
| ABAC 评估错误 | counter | > 阈值告警 |

### 仪表盘分租户

- 按 `TenantId` 分面板（大租户单独），可快速定位某租户异常
- 权限变更事件时间线叠加到业务指标图

## 回滚策略

| 场景 | 回滚方式 |
|------|----------|
| 代码缺陷 | 旧版本镜像重部署 |
| Flag 引发故障 | 关闭 Flag（秒级） |
| DB schema 兼容 | 关闭读新 Flag，下次发布移除变更 |
| DB schema 不兼容 | 执行 down.sql，回滚代码 |
| 租户数据损坏 | 租户级备份恢复（RPO ≤ 1h） |

**强制**：每次发布前必须演练一次回滚脚本。

## 生产数据访问合规

- 生产 DB 连接走堡垒机 + VPN + MFA
- 查询生产敏感数据需 DBA + 安全两级审批
- 所有生产查询记录到只读审计库（包括 SELECT）
- 禁止 `SELECT * FROM Users` 类查询（DBA 在堡垒机层拦截）

## 租户生命周期

| 阶段 | 动作 |
|------|------|
| 开通 | 创建 TenantId、初始化角色/菜单/配置、写审计 |
| 正常 | 按计划支付 / 使用 |
| 欠费宽限 | 只读模式（移除写权限） |
| 停用 | 拒绝登录，保留数据 90 天 |
| 注销 | 导出数据包交付租户，按合规期保留审计，其他硬删除 |

规则：
- 租户注销走工单 + 多级审批 + 两段式确认
- 硬删除脚本必须带 `WHERE TenantId = ?` 且 DBA 复核

## DevOps 评审清单

```
- [ ] 特性开关默认关闭，配置入仓
- [ ] 灰度计划与阶段门禁明确
- [ ] DB 迁移含 up/down/verify 且演练
- [ ] 回滚脚本在 staging 演练通过
- [ ] 监控面板与告警阈值落地
- [ ] 审计写入链路稳定（失败降级 + 补偿）
- [ ] 生产访问走堡垒机 + 审批
- [ ] 租户生命周期操作均在工单系统中留痕
```

## 常见 DevOps 反模式

- ❌ 手工改生产配置表，无审计无回滚
- ❌ DB 迁移与代码发布同一分钟（耦合，难回滚）
- ❌ Flag 永远留在代码里无人清理，累积为"坟场"
- ❌ 监控只有 QPS / 延迟，无授权失败率与隔离指标
- ❌ 发布前从不演练回滚
- ❌ 租户注销用 `DELETE FROM ... WHERE TenantId=X` 直连执行
